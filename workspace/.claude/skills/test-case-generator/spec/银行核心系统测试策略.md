# 银行核心系统测试策略

## 系统特点

银行核心系统是银行业务的中枢，处理账户管理、存款、贷款、支付结算等关键业务，具有高并发、高可用、强一致性的要求。

## 测试重点

### 1. 数据一致性测试（最高优先级）

- **账务一致性**：借方金额合计 = 贷方金额合计
- **总分核对**：总账余额与分户账余额一致
- **日终批处理**：日终批处理结果与日间交易汇总一致
- **跨系统对账**：核心系统与外围系统数据一致

### 2. 并发控制测试

- **并发开户**：多用户同时开立同一类型账户
- **并发转账**：同一账户并发转账场景
- **并发存取款**：同一账户并发存取款场景
- **分布式事务**：跨节点事务的一致性

### 3. 异常场景测试

- **系统中断**：交易处理中系统宕机，数据回滚验证
- **网络超时**：交易提交后网络超时，账务状态验证
- **重复提交**：同一交易重复提交的处理
- **日切处理**：日切时点的交易处理

### 4. 边界值测试

- **金额边界**：0元、1元、单笔限额、日累计限额
- **账户状态**：正常、冻结、止付、销户状态下的交易
- **时间边界**：营业时间内、营业时间外、节假日

## 测试类型权重

| 测试类型 | 权重 | 说明 |
|---------|------|------|
| 功能测试 | 40% | 核心功能正确性 |
| 性能测试 | 20% | 交易响应时间、TPS |
| 数据一致性测试 | 20% | 账务平衡、总分核对 |
| 异常/容错测试 | 15% | 故障恢复、数据回滚 |
| 安全测试 | 5% | 权限控制、数据加密 |

## 必测场景清单

### 存款业务
- [ ] 活期存款开户、销户
- [ ] 定期存款开户、提前支取、到期转存
- [ ] 通知存款建立、取消、支取
- [ ] 存款利息计算（各种计息方式）
- [ ] 存款账户冻结、解冻、止付、解止付

### 贷款业务
- [ ] 贷款发放
- [ ] 贷款正常还款、提前还款
- [ ] 贷款逾期处理
- [ ] 贷款利息计算（正常息、罚息、复利）
- [ ] 贷款展期、核销

### 支付结算
- [ ] 行内转账（本外币）
- [ ] 跨行转账（大小额、超级网银）
- [ ] 批量代发代扣
- [ ] 票据业务（支票、汇票、本票）

### 日终批处理
- [ ] 日终账务平衡检查
- [ ] 利息计提
- [ ] 报表生成
- [ ] 数据备份

## 风险点提示

1. **资金风险**：任何涉及资金变动的测试必须在测试环境执行，禁止操作生产环境
2. **数据风险**：使用脱敏测试数据，不得使用真实客户信息
3. **时间风险**：日终批处理测试需协调时间窗口
4. **依赖风险**：核心系统测试依赖外围系统配合时，需确认接口版本一致

## 性能指标参考

| 指标 | 目标值 |
|-----|--------|
| 单笔交易响应时间 | <= 500ms |
| 日终批处理时间 | <= 2小时 |
| 系统可用率 | >= 99.99% |
| 数据一致性 | 100% |

## 优先级判定参考

| 优先级 | 典型场景 |
|--------|---------|
| 高 | 开户、销户、转账、存取款、贷款发放/还款、日终批处理、利息计算 |
| 中 | 账户查询、交易明细查询、账户状态变更（冻结/解冻）、报表查询 |
| 低 | 统计报表导出、非核心参数配置、提示信息优化 |
