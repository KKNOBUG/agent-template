# CSV 输出格式定义

两个输出文件 `recommended_cases.csv` 和 `ai_cases.csv` 使用相同的列结构，以 `;` 分隔，UTF-8 编码，带 BOM 头。

## 列定义

| 列名 | 说明 | 来源/取值规则 |
|---|---|---|
| `projectAiCodeTestScriptId` | 项目AI代码测试脚本ID | 回归案例：保持与原案例一致；AI补充案例：留空 |
| `testCaseNumber` | 用例编号 | 回归案例：保持原编号；AI补充案例：`AI_001`、`AI_002`... 递增 |
| `testCaseName` | 用例名称 | 回归案例：保持原名称；AI补充案例：描述测试场景的名称 |
| `applicationModel` | 应用模块/系统 | 回归案例：保持原模块；AI补充案例：根据修改涉及的包/类推断 |
| `priority` | 优先级 | `3`=高, `2`=中, `1`=低。回归案例：保持原优先级；AI补充案例：根据影响范围判断 |
| `testCaseType` | 用例类型 | `1`=功能测试, `2`=接口测试, `3`=异常测试, `4`=集成测试。回归案例：保持原类型；AI补充案例：根据测试场景判断 |
| `feedbackStatus` | 反馈状态 | `1`=已采纳, `2`=未采纳, `3`=未标记。回归案例：保持原状态；AI补充案例：默认 `3`(未标记) |
| `type` | 案例类别 | `1`=回归列表, `2`=补充列表。回归案例：固定 `1`；AI补充案例：固定 `2` |

## 优先级判断规则

- `3`(高)：修改的是控制器/接口层、公共工具类，或影响范围大且核心业务相关
- `2`(中)：修改的是服务层，影响范围中等
- `1`(低)：修改的是数据访问层或边缘辅助方法，影响范围较小

## 用例类型判断规则

- `1`(功能测试)：测试正常业务流程和功能点
- `2`(接口测试)：测试 API 接口的输入输出、参数校验
- `3`(异常测试)：测试异常分支、错误处理、边界条件
- `4`(集成测试)：测试跨模块/跨服务的集成场景

## CSV 格式要求

- 分隔符：`;`
- 编码：UTF-8 with BOM (`﻿`)
- 换行：Unix 风格 `\n`
- 字段值包含 `;` 或换行时，用 `"` 包裹整个字段
- 表头行必须存在且列名顺序固定：`projectAiCodeTestScriptId;testCaseNumber;testCaseName;applicationModel;priority;testCaseType;feedbackStatus;type`
